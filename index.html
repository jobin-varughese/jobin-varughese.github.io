Name: Jobin Varughese

Why I’m in this class: I’m taking this class to better understand how people coordinate work in real settings, especially within educational contexts, when collaboration spans between tools and learning systems. I’ve also been following Computer supported collaborative Learning (CSCL) research and hope to publish in that venue. I’m interested in the tradeoffs tools create (speed vs. clarity, engagement vs. outcomes, visibility vs. overload) and how design choices shape group behavior in academic contexts.

Collaborative Software I Use Regularly

  • Friends & family: I primarily use SMS/iMessage for domestic and WhatsApp for international, FaceTime
  • Meetings: Outlook calendar, Google Meet, Zoom, Microsoft Teams (This is my everyday as I work full-time)
  • Work coordination: Email, Canvas
  • Collaborative writing: Google Docs, Microsoft Word/OneDrive
  • Project coordination: GitHub, Asana, TeamDynamix
  • Scheduling: Outlook Calendar

What Helps Most (and What Could Be Better)

Features that help the most:

  • I find the shared visibility (calendars, shared docs, task lists) and awareness reduces ambiguity about who is doing what.
  • Version history (Docs/GitHub) makes changes trackable and reversible.
  • Asynchronous communication supports collaboration across different schedules.
  • Screen sharing helps align quickly when a problem is easier to show than explain. This is probably my biggest takeaway in CSCW work as I have done multiple trainings virtually.

What could be better:

  • Notification overload across tools makes it easy to miss important information.
  • Context gets fragmented across chat, email, and documents. Sometimes, in spite of collaboration, tool overload could become cognitive overload.
  • It can be hard to reconstruct the “why” behind decisions later.
